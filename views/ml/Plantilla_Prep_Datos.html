Tratamiento de NAs

En todo dataset o en toda base de datos encontraremos ciertos datos faltantes en este caso no es la excepción, hay módulos como scikit Learn que nos proporcionan métodos para corregir eso y que al mandar los datos a un modelo este no tenga errores o los resultados sean inexactos.

SimpleImputer esta función remplaza los valores faltantes calculando la mediana o media de toda la columna (Leer documentación)

from sklearn.impute import SimpleImputer

imputer = SimpleImputer(missing_values=np.nan, strategy='mean')

imputer.fit(X[:,0:2])

X[:,0:2] = imputer.transform(X[:,0:2])

El reemplazo se aplicará a X, aclarando que se está leyendo y sobrescribiendo la tabla X, ya que el dataset ya no lo volveremos a tocar

Leer la documentación y diferencia entre fit(), transform() y fit_transform() ya que suele ser algo confuso y estas funciones se volverán a usar más adelante

A como yo entendí:

Fit: ajusta los valores, en este caso la media y los guarda como objetos internos.

Transform: los parámetros hechos con el método fit los usa y los aplica a los datos para actualizarlos.

Fit_transform: ajusta y transforma los valores simultáneamente.

Codificar los datos categóricos

Los datos categóricos se deben transformar a números para poder trabajar en Python con ellos. En este caso las columnas son Country y Purchased ya son strings. El módulo de scikit Learn con su función LabelEncoder (Leer documentación) nos ayuda en ese aspecto.

from sklearn.preprocessing import LabelEncoder, OneHotEncoder

from sklearn.compose import ColumnTransformer

le_X =LabelEncoder()

X[:,0]=le_X.fit_transform(X[:,0])

De las variables independientes la columna Country sobrescribirá dicha columna por valores enteros, a las que tengan la misma ciudad tendrán el mismo valor entero.

Transformación de la columna Country a variables dummy, Crenado una columna por cada categoría y de las variables dummy solo se marca la categoría correcta con un booleano.

Para ello nos servirá la función ColumnTransform y el transformador OneHotEncoder (Leer documentación)

ct = ColumnTransformer(

# [(‘nombre’,transformador,columna a alterar)]

[('one_hot_encoder',OneHotEncoder(categories='auto',dtype=int),[0])],

remainder='passthrough') # el resto de columnas se transladan sin alterarse

X = np.array(ct.fit_transform(X),dtype=float)

Tratamiento de NAs

La función que usamos para el tratamiento de NAs en Python trabaja sacando un promedio de toda la columna y dicho promedio lo pone en donde hay un valor faltante, en R no hay una librería que haga eso por nosotros, asi que lo hacemos de la siguiente forma.

dataset$Age = ifelse(is.na(dataset$Age),

# El valor que se va a sustituir que sera la media

ave(dataset$Age,FUN= function(x) mean(x,na.rm=TRUE)),

# se deja el valor como esta por defecto

dataset$Age)

dataset$Salary = ifelse(is.na(dataset$Salary),

# El valor que se va a sustituir que sera la media

ave(dataset$Salary,FUN= function(y) mean(y,na.rm=TRUE)),

# se deja el valor como esta por defecto

dataset$Salary)

Ver la documentación de if - else en R.

En split almacenaremos un arreglo con valores booleanos, los TRUE serán del 80% y el resto serán FALSE y con ello se seleccionan los conjuntos de test y de entrenamiento.

# configurando semilla aleatoria para la division de datos

set.seed(10)

# se elige el porcentaje de los datos para el training en %

split = sample.split(dataset$Purchased,SplitRatio = 0.8)

print(split)

# Dividiendo el conjunto , False para el test

training_set = subset(dataset,split == FALSE)

# Dividiendo el conjunto , True para el training

testing_set = subset(dataset,split == TRUE)
